Comparison between plain objects repeated and using a class once

Without class — BAD approach
// Make 3 students manually
// Copy-paste same structure 3 times!

const s1 = {
  name:"Ali", age:22,
  greet(){return "Hi "+this.name;}
};
const s2 = {
  name:"Sara", age:21,
  greet(){return "Hi "+this.name;}
};
const s3 = {
  name:"Raj", age:23,
  greet(){return "Hi "+this.name;}
};

// 1000 students = 1000 copies of greet!
✕ greet() copied 3 times
✕ Change means fixing everywhere
✕ Not scalable at all
✕ Inconsistent structure
With class — GOOD approach
// Write template ONCE
class Student {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  greet() {
    return "Hi " + this.name;
  }
}

// Create 1000 students in 3 lines!
const s1 = new Student("Ali", 22);
const s2 = new Student("Sara", 21);
const s3 = new Student("Raj", 23);
✓ greet() defined once, used by all
✓ Change in one place — fixes all
✓ Scale to 1 million students easily
✓ Guaranteed same structure every time
