Getters setters and static methods in JavaScript classes

// get — access like a property but runs a function
class Student {
  constructor(name, score) {
    this.name = name;
    this.score = score;
  }
  get grade() { // no () when you access it!
    return this.score >= 90 ? "A" : this.score >= 75 ? "B" : "C";
  }
  get summary() {
    return `${this.name} → grade ${this.grade}`;
  }
}

const ali = new Student("Ali", 88);
ali.grade // "B" — no () needed! looks like a property
ali.summary // "Ali → grade B"
88
ali.grade → "B"  |  ali.summary → "Ali → grade B"
Why getters? Computed values that look like properties. No parentheses needed when accessing — cleaner code!
// set — validate before storing a value
class Student {
  constructor(name) {
    this.name = name;
    this._score = 0; // private by convention (_)
  }
  get score() { return this._score; }
  set score(v) {
    if (v < 0 || v > 100) throw "Score must be 0-100!";
    this._score = v;
  }
}

ali.score = 88; // ✓ valid — stored
ali.score = 200; // ✕ Error: Score must be 0-100!
—
Why setters? Validate data BEFORE it's stored. Prevents invalid values entering your object.
// static — belongs to the CLASS, not instances
class MathHelper {
  static add(a, b) { return a + b; }
  static multiply(a, b) { return a * b; }
  static PI = 3.14159;
}

// Call directly on the CLASS — no new needed!
MathHelper.add(5, 3) // 8
MathHelper.multiply(4, 5) // 20
MathHelper.PI // 3.14159

// ✕ Cannot call on instance:
const m = new MathHelper();
m.add(1, 2); // TypeError: m.add is not a function
—
Why static? Utility functions that don't need object data. Like Math.round(), Math.max() — you call them on Math directly, not on an instance.
